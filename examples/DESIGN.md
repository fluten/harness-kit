# Part 1：视觉规范

## 设计风格

简洁专业，数据密度适中，参考 Linear / Vercel Dashboard 风格。面向 B2B 企业用户，克制而不冷漠。

## 色彩体系

### 主色
| 用途 | 色值 | 使用场景 |
|------|------|----------|
| Primary | #2563EB (Blue-600) | 主要按钮、重点强调、导航选中态 |
| Secondary | #64748B (Slate-500) | 次要操作、辅助信息 |

### 中性色
| 用途 | 色值 | 使用场景 |
|------|------|----------|
| Background | #F8FAFC (Slate-50) | 页面底色 |
| Surface | #FFFFFF | 卡片/面板底色 |
| Border | #E2E8F0 (Slate-200) | 分割线、边框 |
| Text Primary | #0F172A (Slate-900) | 正文 |
| Text Secondary | #64748B (Slate-500) | 次要文字、说明 |

### 语义色
| 用途 | 色值 | 使用场景 |
|------|------|----------|
| Success / Positive | #16A34A (Green-600) | 正面情绪标签、成功状态 |
| Warning | #D97706 (Amber-600) | 警告状态 |
| Error / Negative | #DC2626 (Red-600) | 负面情绪标签、错误状态 |
| Info / Neutral | #64748B (Slate-500) | 中性情绪标签、提示信息 |

### 深色模式
MVP 阶段暂不支持深色模式，后续版本通过 CSS 变量切换实现。

## 字体

| 层级 | 字号 | 字重 | 行高 | 使用场景 |
|------|------|------|------|----------|
| H1 | 24px | Bold (700) | 32px | 页面标题 |
| H2 | 20px | Semibold (600) | 28px | 区块标题、卡片标题 |
| H3 | 16px | Medium (500) | 24px | 小标题 |
| Body | 14px | Regular (400) | 20px | 正文 |
| Caption | 12px | Regular (400) | 16px | 辅助说明、时间戳 |

- 中文字体：system-ui, "PingFang SC", "Microsoft YaHei"
- 英文字体：Inter, system-ui
- 等宽字体（数据/代码）：JetBrains Mono, monospace

## 间距系统

| 名称 | 值 | Tailwind 对应 | 使用场景 |
|------|-----|---------------|----------|
| xs | 4px | p-1 / m-1 | 紧凑元素内间距 |
| sm | 8px | p-2 / m-2 | 元素内间距、标签间距 |
| md | 16px | p-4 / m-4 | 卡片内间距、区块间距 |
| lg | 24px | p-6 / m-6 | 大区块间距 |
| xl | 32px | p-8 / m-8 | 页面级间距、侧边栏与内容区间距 |

## 圆角
| 类型 | 值 | 使用场景 |
|------|-----|----------|
| 小 | 6px | 按钮、输入框、标签 |
| 中 | 8px | 卡片、面板 |
| 大 | 12px | 弹窗、模态框 |

## 组件规范

### 按钮
| 类型 | 样式 | 使用场景 |
|------|------|----------|
| Primary | 蓝底白字、hover 加深 | 主要操作（每屏最多 1 个） |
| Secondary | 白底蓝字蓝边 | 次要操作 |
| Ghost | 无边框、hover 灰底 | 低优先级操作 |
| Danger | 红底白字 | 危险操作（删除等） |

### 情感标签（ChatLens 专用）
| 类型 | 样式 | Tailwind |
|------|------|----------|
| Positive | 绿底绿字、圆角 pill | bg-green-50 text-green-700 |
| Neutral | 灰底灰字、圆角 pill | bg-slate-100 text-slate-600 |
| Negative | 红底红字、圆角 pill | bg-red-50 text-red-700 |

### 输入框
- 默认高度：36px
- 边框样式：1px solid Slate-200
- 聚焦态：边框变 Primary + 外发光
- 错误态：边框变 Error + 下方红色提示文字

### 数据卡片（看板用）
- 白色背景、8px 圆角、轻阴影（shadow-sm）
- 顶部：指标名（Caption 字号）+ 趋势箭头
- 中部：大数字（H1 字号、Primary 色或语义色）
- 底部：对比文字（Caption 字号、灰色）

# Part 2：交互设计

## 页面结构

- 整体布局：左侧固定侧边栏（220px）+ 右侧内容区（flex-1, 可滚动）
- 侧边栏包含：Logo, 导航菜单（看板/对话/导入）, 底部用户信息
- 内容区顶部：Topbar（面包屑 + 时间范围筛选 + 用户头像）
- 内容区主体：根据当前路由渲染对应页面

## 页面流

- 未登录用户：访问任何页面 → 重定向到登录页
- 登录成功 → 看板页（首页）
- 看板页 → 可下钻到对话列表（按条件筛选）
- 对话列表 → 点击单条 → 对话详情页
- 侧边栏导航 → 数据导入页
- 导入页：上传 CSV → 预览 → 确认导入 → 导入成功 → 可触发分析

## 状态与反馈

### 加载态
- 全页加载：居中 Spinner + "加载中..."
- 局部加载：卡片内 Skeleton（骨架屏）
- 骨架屏：是（看板卡片、对话列表）

### 空状态
- 无数据时显示：插画 + "还没有数据，上传对话开始分析" + 上传按钮

### 错误处理
- 网络错误：Toast 提示 "网络异常，请重试"
- 表单校验：输入框下方红色文字
- 权限不足：跳转登录页

## 动效规范
| 类型 | 时长 | 缓动函数 | 使用场景 |
|------|------|----------|----------|
| 入场 | 200ms | ease-out | 页面切换、弹窗打开 |
| 退场 | 150ms | ease-in | 弹窗关闭、元素移除 |
| 交互反馈 | 100ms | ease-in-out | hover、点击 |
| 数据更新 | 300ms | ease-out | 图表数据切换、数字变化 |

## 响应式

MVP 只做桌面端（≥ 1024px）。面试演示场景基本都是电脑。

## 规则
1. 所有颜色使用 Tailwind CSS 的预设色值或 CSS 变量，禁止硬编码 hex
2. 间距只使用间距系统中定义的值（对应 Tailwind 的 p-1/p-2/p-4/p-6/p-8）
3. 新增组件前先检查是否已有可复用的组件
4. 动效时长不超过 300ms，避免拖慢交互感知
5. 情感标签的颜色必须和语义色保持一致（正面=绿、负面=红、中性=灰）
6. 视觉规范变更需同步检查已有组件是否受影响
