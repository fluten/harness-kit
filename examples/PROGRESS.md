### 📊 快速状态 重建算法

修改本文件 body（项目状态 / 最近完成 / 当前阻塞 / 里程碑 / Session 日志）任一区块后，必须按以下步骤**整体重写**顶部「📊 快速状态」区块（禁止增量修改字段）。

**输入**：当前 PROGRESS.md 的 body 部分
**输出**：5 行的「📊 快速状态」块

**步骤**：
1. `PHASE` = 「## 项目状态：xxx」标题中冒号后的文本（如 `🟡 规划中`、`🔵 开发中`）
2. `MILESTONE` 取值规则：
   - 扫描「## 里程碑」下所有 `### M*` 子标题
   - 取第一个状态非 `✅已达成` 的里程碑标题（去掉 `### ` 前缀）
   - 若全部已达成 → `MILESTONE` = `全部达成`
3. `PROGRESS_PCT` 取值规则：
   - 在 `MILESTONE` 对应里程碑的"包含功能"列表中数总功能数 `N_total`
   - 数已完成（标记 ✅ 或文本含"已完成"）的功能数 `N_done`
   - `PROGRESS_PCT` = `round(N_done / N_total * 100)%`
   - 若 `N_total = 0` → `PROGRESS_PCT` = `0%`
4. `N_block` = 「## 当前阻塞」区下条目（一行一项）的数量
5. `NEXT_FROM` 取值规则：
   - 若「## Session 日志」非空 → 取最新 `### Session YYYY-MM-DD` 下的「下次继续」字段值
   - 否则 → 读 TODO.md「## 当前任务」首项的描述
   - 都没有 → `NEXT_FROM` = `（未开始）`
6. `DATE` = 今天的日期（`YYYY-MM-DD` 绝对格式）

**填入顶部「📊 快速状态」区块**（按以下模板原样输出，将 `{var}` 替换为算出的值）：

````markdown
- **阶段**: `{PHASE}`
- **当前里程碑**: `{MILESTONE}`（进度 `{PROGRESS_PCT}`）
- **阻塞计数**: `{N_block}` 项
- **下次从**: `{NEXT_FROM}`
- **更新**: `{DATE}`
````
