## 项目概述
- 项目名：ChatLens
- 目标用户：香港中小企业的客服/销售团队负责人、运营经理
- 核心价值：从 AI 客服对话中自动提取质量洞察、情绪趋势和高频问题，替代人工抽检
- 产品形态：Web 应用（SaaS）

## 功能模块

### 模块一：认证登录（AUTH）
- **功能描述**：企业账号注册、登录、权限管理
- **用户故事**：作为企业管理员，我希望用邮箱注册并邀请团队成员，以便团队共同使用分析平台
- **核心交互**：注册 → 邮箱验证 → 登录 → 邀请成员
- **输入/输出**：邮箱+密码 → JWT Token + 用户角色
- **边界条件与异常处理**：邮箱重复注册、Token 过期刷新、密码强度校验

### 模块二：数据导入（IMP）
- **功能描述**：支持上传 CSV/JSON 格式的对话数据，或通过 API 接入实时对话流
- **用户故事**：作为运营经理，我希望一键上传历史对话记录，以便快速看到分析结果
- **核心交互**：选择文件 → 字段映射 → 数据预览 → 确认导入
- **输入/输出**：CSV/JSON 文件 → 结构化对话记录入库
- **边界条件与异常处理**：文件格式错误、字段缺失、重复导入检测、大文件分片上传

### 模块三：对话分析（CONV）
- **功能描述**：对导入的对话进行 NLP 分析，包括情感分析、意图分类、关键词提取、质量评分
- **用户故事**：作为客服主管，我希望自动识别每通对话的情绪走向和客户意图，以便快速定位问题对话
- **核心交互**：选择数据集 → 触发分析 → 查看单条对话详情 → 筛选/排序
- **输入/输出**：原始对话文本 → 情感标签 + 意图标签 + 关键词 + 质量评分
- **边界条件与异常处理**：粤语/中英混排文本处理、分析超时、LLM API 调用失败重试

### 模块四：数据看板（DASH）
- **功能描述**：可视化展示对话分析的汇总数据，包括情绪趋势、高频问题 Top N、质量分布、客服排名
- **用户故事**：作为企业负责人，我希望打开看板就能看到本周的客服质量概览，以便做管理决策
- **核心交互**：选择时间范围 → 查看趋势图/排名表 → 下钻到具体对话
- **输入/输出**：时间范围 + 筛选条件 → 图表 + 统计数据
- **边界条件与异常处理**：数据为空时的空状态、大数据量下的分页和性能

### 模块五：报告生成（RPT）
- **功能描述**：一键生成周报/月报 PDF，包含关键指标摘要和趋势分析
- **用户故事**：作为运营经理，我希望每周自动生成一份质检报告发给老板，以便展示工作成果
- **核心交互**：选择报告类型和时间范围 → 预览 → 导出 PDF
- **输入/输出**：时间范围 + 报告模板 → PDF 文件
- **边界条件与异常处理**：无数据时期的报告处理、PDF 生成超时

## 非功能需求
- 性能要求：看板页首屏加载 < 2s，对话分析单条 < 5s
- 兼容性：Chrome / Edge / Safari 最新版本，桌面端优先
- 安全性：密码 bcrypt 加密、API 全程 HTTPS、JWT 鉴权
- 可访问性：MVP 阶段不做特别处理，后续版本补充

## 数据需求
核心数据实体关系：Organization 1:N User, Organization 1:N Dataset, Dataset 1:N Conversation, Conversation 1:N Message, Conversation 1:1 Analysis。详见 SCHEMA.md。

## 技术约束
- 前端：React 18 + TypeScript + Tailwind CSS + Zustand + Recharts
- 后端：Python 3.11 + FastAPI + SQLAlchemy + Alembic
- 数据库：PostgreSQL 15
- NLP：调用 LLM API（DeepSeek / Claude）做分析，不自建模型
- 部署：Docker Compose 本地可跑，演示用 Vercel（前端）+ Railway（后端）

## MVP 范围

### 包含
- 用户注册/登录（单企业，不做多租户）
- CSV 文件上传导入
- 对话情感分析 + 意图分类（调用 LLM API）
- 数据看板（情绪趋势、高频问题、质量分布）
- 单条对话详情查看

### 不包含（后续版本）
- API 实时接入
- 多租户/权限管理
- 报告生成 PDF 导出
- 粤语语音转文字
- 自定义分析模型
- 移动端适配

## 术语表
| 术语 | 定义 |
|------|------|
| 对话（Conversation） | 一次完整的客服/销售交互，包含多条消息 |
| 消息（Message） | 对话中的单条发言，含发言者角色（客服/客户） |
| 情感标签（Sentiment） | 对话或消息的情绪倾向：正面/中性/负面 |
| 意图标签（Intent） | 客户发起对话的目的分类：咨询/投诉/退换/购买等 |
| 质量评分（Quality Score） | 客服对话质量的综合打分（0-100） |
| 数据集（Dataset） | 一次导入的对话数据集合 |
